     Give main_imgui.cpp full feature parity with main_sdl.c

     Context

     main_imgui.cpp is currently just the unmodified Dear ImGui
     SDL3+SDL_Renderer example (a demo window toggle and empty menu stubs).
     main_sdl.c is the real emulator frontend: it parses config/CLI args, loads
     a ROM, wires up audio, movie playback, configurable keybindings, debug
     RAM/PPU dump hotkeys, an OpenGL tile renderer (SMBgl) or a software raster
     renderer (SMBraster), and runs the game loop at NES framerate (or uncapped
     in "maxspeed" mode). The goal is to make main_imgui.cpp do everything
     main_sdl.c does, but with the Dear ImGui menu bar overlaid on top of the
     running game, per user decisions:
     - Support both the OpenGL (SMBgl) and software raster (SMBraster) emulator
     renderers, selected the same way as today (cfg.graphics.opengl /
     cfg.graphics.software).
     - The game picture fills the whole window as a background, letterboxed to
     preserve the 256:240 aspect ratio, with the ImGui menu bar drawn on top.
     - Full feature parity: same config/ini parsing, CLI args, keybindings,
     audio, movie playback, debug dump hotkeys, smb2j games-beaten persistence,
     and maxspeed mode.

     Key architectural change: swap SDL_Renderer for OpenGL

     main_imgui.cpp currently uses imgui_impl_sdlrenderer3 (SDL's own 2D
     renderer). To draw the SMBgl renderer's raw GL calls into the same window
     Dear ImGui draws into, the frontend needs an actual OpenGL context. Switch
     to imgui_impl_opengl3 (already vendored in imgui/backends/), following the
     pattern in imgui/examples/example_sdl3_opengl3/main.cpp:
     - SDL_GL_SetAttribute for a 3.3 core profile context (matches
     render_opengl.c's #version 330 shaders), depth buffer 24-bit (used for
     BG/sprite ordering).
     - ImGui_ImplSDL3_InitForOpenGL(window, gl_context) +
     ImGui_ImplOpenGL3_Init().

     For raster mode, upload the SMBraster RGB buffer into a plain GLuint
     texture each frame (glTexSubImage2D) and draw it via
     ImGui::GetBackgroundDrawList()->AddImage(...) ‚ÄĒ no flip needed; feeding the
     texture uv range (0,0)-(1,1) directly maps row 0 (top of the NES picture)
     to the top of the destination rect, matching how render_opengl.c's own
     pattern-table texture is sampled (no flip there either).

     Letterboxing math

     Add a small helper compute_letterbox(avail_w, avail_h, &x, &y, &w, &h) that
     fits a 256:240 rect centered within the available area. Use it twice per
     frame, in two different coordinate spaces:
     - Framebuffer pixels (SDL_GetWindowSizeInPixels) ‚ÄĒ for the raw
     glViewport/glScissor calls that constrain the SMBgl GL renderer to the
     letterboxed region. GL's viewport/scissor origin is bottom-left, so
     convert: gl_y = fb_h - (ly + lh).
     - ImGui logical/display coordinates (io.DisplaySize) ‚ÄĒ for the AddImage
     destination rect in raster mode (ImGui's draw list is already top-left
     origin, no conversion needed).

     Each frame: clear the entire window to black first (so letterbox bars are
     consistent for both renderer paths), then draw the letterboxed content:
     - GL mode: glEnable(GL_SCISSOR_TEST), glScissor+glViewport to the letterbox
     rect, call SMB_tick + SMBgl_render_frame (its internal glClear/draw stays
     confined to the scissor rect), then glDisable(GL_SCISSOR_TEST).
     - Raster mode: SMBraster_set_buffer/SMBraster_clear into a heap-allocated
     256*240*3 RGB buffer owned by frontend_userdata, SMB_tick, upload via
     glTexSubImage2D, then ImGui::GetBackgroundDrawList()->AddImage into the
     letterboxed rect (drawn before ImGui::Render() so it lands behind the menu
     bar).

     This intentionally does not touch render_opengl.c/render_raster.c ‚ÄĒ both
     are reused unmodified via their existing SMBgl_*/SMBraster_* APIs.

     main_imgui.cpp rewrite

     Mirror main_sdl.c's structure closely (per its own "one unashamedly-long
     main, keep it simple" convention) ‚ÄĒ duplicate rather than extract shared
     helpers, matching the project's existing single-file-per-frontend style:

     - Copy over verbatim: io_readtobuffer/io_writefrombuffer, joy1/joy2,
     update_pattern_tables/update_palette/draw_tile,
     load_palette/load_ppuram/dump_ppuram/load_ram/dump_ram,
     smb2j_load_games_beaten/smb2j_save_games_beaten, advance_movie,
     on_keypress_change, read_rom_bytes/seek_rom,
     apu_write_register/apu_end_frame, and #include "config.h" (config.h is
     plain C, compiles fine as C++; already used this way conceptually since
     audio.h/movie.h/timer.h all have extern "C" guards for that purpose).
     - frontend_userdata gains: GLuint raster_texture, unsigned char 
     *raster_pixels (in place of the SDL_Surface *surf/pixels_stride used for
     the SDL_Renderer path).
     - Replace sdl_tick with imgui_tick(void *userdata), used identically by
     both timer_run_at_frequency (capped fps) and the maxspeed while(1) loop ‚ÄĒ
     same as main_sdl.c. It does, in order:
       a. Poll SDL events: forward every event to ImGui_ImplSDL3_ProcessEvent;
     keep the existing SDL_EVENT_KEY_DOWN/UP ‚Üí on_keypress_change handling and
     SDL_EVENT_QUIT/SDL_EVENT_WINDOW_CLOSE_REQUESTED ‚Üí return 1 (no
     WantCaptureKeyboard gating ‚ÄĒ matches main_sdl.c, which always routes keys
     to the game).
       b. advance_movie(fe).
       c. If minimized, SDL_Delay(10) and return 0 (skip rendering), matching
     the original demo's behavior.
       d. ImGui_ImplOpenGL3_NewFrame() / ImGui_ImplSDL3_NewFrame() /
     ImGui::NewFrame().
       e. Full-window black clear, then the letterboxed GL-or-raster draw
     described above.
       f. Build the menu bar (kept from the existing stub: File > Load ROM stays
     a no-op placeholder ‚ÄĒ main_sdl.c has no runtime ROM-switching either, so
     this isn't a feature gap; Debug > Show ImGui demo window).
       g. ImGui::Render(), ImGui_ImplOpenGL3_RenderDrawData, SDL_GL_SwapWindow.
     - main() follows main_sdl.c's structure (config init ‚Üí defaults ‚Üí
     parse_options("smbvanilla.ini", ...) ‚Üí validate ROM arg ‚Üí keybindings from
     cfg.bindings.keys ‚Üí audio init ‚Üí palette load ‚Üí window/GL-context creation
     (always, since ImGui itself needs a GL context) ‚Üí SMBgl_init if
     cfg.graphics.opengl else SMBraster_init if cfg.graphics.software ‚Üí allocate
     raster_texture/raster_pixels if raster ‚Üí ROM
     open/SMB_state_init/SMB_start_on_level ‚Üí movie load + skip-to-frame ‚Üí
     fe->rendering_enabled = true ‚Üí run loop (imgui_tick via
     timer_run_at_frequency or uncapped) ‚Üí cleanup (mirrors main_sdl.c's exit:
     label, plus
     ImGui_ImplOpenGL3_Shutdown/ImGui_ImplSDL3_Shutdown/ImGui::DestroyContext,
     freeing raster_pixels/deleting raster_texture, guarded by an
     imgui_initialized bool since backend init happens after window creation
     which can fail).
     - Window title becomes "SMB Vanilla"; initial size
     256*video_scale*main_scale x 240*video_scale*main_scale; keep
     SDL_WINDOW_RESIZABLE.
     - Respect cfg.general.maxspeed for both the uncapped loop and
     SDL_GL_SetSwapInterval(0).

     Build system changes

     - meson.build (root): the shared imgui library currently compiles
     imgui/backends/imgui_impl_sdlrenderer3.cpp; swap it for
     imgui/backends/imgui_impl_opengl3.cpp (SDL3 platform backend stays as-is).
     - src/platform/meson.build: the smbvanilla_imgui target currently only

     - No changes to smbvanilla.ini ‚ÄĒ the imgui frontend reads the same config
     file, matching parity intent.
     - Note (not fixed, pre-existing/out of scope): the shared imgui library

     smbvanilla_imgui only really supports SDL3 today ‚ÄĒ same limitation that
     exists before this change.
